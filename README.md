# Интернет-магазин «Web-Larёk»

«Web-Larёk» — это интернет-магазин с товарами для веб-разработчиков, где пользователи могут просматривать товары, добавлять их в корзину и оформлять заказы. Сайт предоставляет удобный интерфейс с модальными окнами для просмотра деталей товаров, управления корзиной и выбора способа оплаты, обеспечивая полный цикл покупки с отправкой заказов на сервер.

## Архитектура приложения

Код приложения разделен на слои согласно парадигме MVP (Model-View-Presenter), которая обеспечивает четкое разделение ответственности между классами слоев Model и View. Каждый слой несет свой смысл и ответственность:

Model - слой данных, отвечает за хранение и изменение данных.  
View - слой представления, отвечает за отображение данных на странице.  
Presenter - презентер содержит основную логику приложения и отвечает за связь представления и данных.

Взаимодействие между классами обеспечивается использованием событийно-ориентированного подхода. Модели и Представления генерируют события при изменении данных или взаимодействии пользователя с приложением, а Презентер обрабатывает эти события используя методы как Моделей, так и Представлений.

### Базовый код

#### Класс Component

Является базовым классом для всех компонентов интерфейса.
Класс является дженериком и принимает в переменной `T` тип данных, которые могут быть переданы в метод `render` для отображения.

Конструктор:  
`constructor(container: HTMLElement)` - принимает ссылку на DOM элемент за отображение, которого он отвечает.

Поля класса:  
`container: HTMLElement` - поле для хранения корневого DOM элемента компонента.

Методы класса:  
`render(data?: Partial<T>): HTMLElement` - Главный метод класса. Он принимает данные, которые необходимо отобразить в интерфейсе, записывает эти данные в поля класса и возвращает ссылку на DOM-элемент. Предполагается, что в классах, которые будут наследоваться от `Component` будут реализованы сеттеры для полей с данными, которые будут вызываться в момент вызова `render` и записывать данные в необходимые DOM элементы.  
`setImage(element: HTMLImageElement, src: string, alt?: string): void` - утилитарный метод для модификации DOM-элементов `<img>`

#### Класс Api

Содержит в себе базовую логику отправки запросов.

Конструктор:  
`constructor(baseUrl: string, options: RequestInit = {})` - В конструктор передается базовый адрес сервера и опциональный объект с заголовками запросов.

Поля класса:  
`baseUrl: string` - базовый адрес сервера  
`options: RequestInit` - объект с заголовками, которые будут использованы для запросов.

Методы:  
`get(uri: string): Promise<object>` - выполняет GET запрос на переданный в параметрах ендпоинт и возвращает промис с объектом, которым ответил сервер  
`post(uri: string, data: object, method: ApiPostMethods = 'POST'): Promise<object>` - принимает объект с данными, которые будут переданы в JSON в теле запроса, и отправляет эти данные на ендпоинт переданный как параметр при вызове метода. По умолчанию выполняется `POST` запрос, но метод запроса может быть переопределен заданием третьего параметра при вызове.  
`handleResponse(response: Response): Promise<object>` - защищенный метод проверяющий ответ сервера на корректность и возвращающий объект с данными полученный от сервера или отклоненный промис, в случае некорректных данных.

#### Класс EventEmitter

Брокер событий реализует паттерн "Наблюдатель", позволяющий отправлять события и подписываться на события, происходящие в системе. Класс используется для связи слоя данных и представления.

Конструктор класса не принимает параметров.

Поля класса:  
`_events: Map<string | RegExp, Set<Function>>)` - хранит коллекцию подписок на события. Ключи коллекции - названия событий или регулярное выражение, значения - коллекция функций обработчиков, которые будут вызваны при срабатывании события.

Методы класса:  
`on<T extends object>(event: EventName, callback: (data: T) => void): void` - подписка на событие, принимает название события и функцию обработчик.  
`emit<T extends object>(event: string, data?: T): void` - инициализация события. При вызове события в метод передается название события и объект с данными, который будет использован как аргумент для вызова обработчика.  
`trigger<T extends object>(event: string, context?: Partial<T>): (data: T) => void` - возвращает функцию, при вызове которой инициализируется требуемое в параметрах событие с передачей в него данных из второго параметра.

### Данные

#### Интерфейс IProduct

Описывает товар, доступный для просмотра и покупки в интернет-магазине.

Поля интерфейса:
`id: string` - идентификатор товара.
`title: string` - наименование товара.
`image: string` - ссылка на изображение товара.
`category: string` - категория товара.
`price: number | null` - стоимость товара.
`description: string` - описание товара.

#### Тип TPayment

Описывает способы, доступные для оплаты.

Значения:
`'card'` - оплата банковской картой.
`'cash'` - оплата наличными.

#### Интерфейс IBuyer

Описывает данные покупателя, необходимые для оформления заказа в интернет-магазине.

Поля интерфейса:
`payment: TPayment | null` - выбранный способ оплаты. До выбора содержит значение null.
`address: string` - адрес доставки.
`email: string` - электронная почта покупателя.
`phone: string` - телефонный номер покупателя.

#### Интерфейс IOrder

Описывает данные заказа, отправляемые на сервер.

Поля интерфейса:
`payment: TPayment` - выбранный способ оплаты.
`address: string` - адрес доставки.
`email: string` - электронная почта покупателя.
`phone: string` - телефонный номер покупателя.
`items: string[]` - массив идентификаторов товаров.
`total: number` - общая стоимость заказа.

#### Интерфейс IProductsResponse

Описывает ответ сервера при запросе списка товаров.

Поля интерфейса:
`total: number` - количество товаров.
`items: IProduct[]` - массив товаров.

#### Интерфейс IOrderResponse

Описывает ответ сервера после оформления заказа.

Поля интерфейса:
`id: string` - идентификатор заказа.
`total: number` - итоговая сумма заказа.

### Модели данных

#### Класс ProductsModel

Отвечает за хранение массива всех товаров, а также товара, выбранного для подробного отображения.

Поля класса:
`items: IProduct[]` - массив всех товаров.
`preview: IProduct | null` - товар, выбранный для подробного отображения.

Методы класса:
`setItems(items: IProduct[]): void` - сохранение массива товаров полученного в параметрах метода.
`getItems(): IProduct[]` - получение массива товаров из модели.
`getProduct(id: string): IProduct | undefined` - получение одного товара по его идентификатору.
`setPreview(product: IProduct): void` - сохранение товара для подробного отображения.
`getPreview(): IProduct | null` - получение товара для подробного отображения.

#### Класс BasketModel

Отвечает за хранение массива товаров, выбранных пользователем для покупки.

Поля класса:
`items: IProduct[]` - массив товаров, выбранных покупателем для покупки.

Методы класса:
`getItems(): IProduct[]` - получение массива товаров, которые находятся в корзине.
`addProduct(product: IProduct): void` - добавление товара в массив корзины.
`removeProduct(id: string): void` - удаление товара из массива корзины.
`clear(): void` - очистка корзины.
`getTotalPrice(): number` - получение стоимости всех товаров в корзине.
`getCount(): number` - получение количества товаров в корзине.
`hasProduct(id: string): boolean` - проверка наличия товара в корзине по его идентификатору.

#### Класс BuyerModel

Отвечает за хранение данных покупателя и их проверку.

Поля класса:
`data: IBuyer` - объект с данными покупателя.

Методы класса:
`setData(data: Partial<IBuyer>): void` - обновляет данные покупателя.
`getData(): IBuyer` - возвращает данные покупателя.
`clear(): void` - очищает данные покупателя.
`validate(): TValidateErrors` - выполняет проверку введенных данных и возвращает сообщения об ошибках.

### Слой коммуникации

#### Класс WebLApi

Отвечает за взаимодействие приложения с сервером.

Конструктор:
`constructor(api: IApi)` - принимает объект, реализующий интерфейс IApi.

Поля класса:
`api: IApi` - экземпляр класса для выполнения HTTP-запросов.

Методы класса:
`getProducts(): Promise<IProductsResponse>` - получает с сервера список товаров.

### Слой представления (View)

#### Класс Page

Отвечает за отображение основных элементов главной страницы.

Поля класса:
`counter: HTMLElement` - элемент отображения количества товаров в корзине.
`gallery: HTMLElement` - контейнер каталога товаров.
`basketButton: HTMLButtonElement` - кнопка открытия корзины.

Методы класса:
`set counter(value: number)` - обновляет счетчик товаров в корзине.
`set gallery(items: HTMLElement[])` - отображает карточки товаров в каталоге.

#### Класс Modal

Отвечает за отображение модального окна.

Конструктор:
`constructor(container: HTMLElement, events: IEvents)` - принимает контейнер модального окна и объект для обработки событий.

Поля класса:
`closeButton: HTMLButtonElement` - кнопка закрытия модального окна.
`content: HTMLElement` - контейнер содержимого модального окна.
`events: IEvents` - объект для обработки событий.

Методы класса:
`open(): void` - открывает модальное окно и генерирует событие открытия.
`close(): void` - закрывает модальное окно, очищает его содержимое и генерирует событие закрытия.
`render(data: { content: HTMLElement }): HTMLElement` - отображает компонент в модальном окне.
`get isOpen(): boolean` - возвращает информацию, открыто ли модальное окно.

#### Класс Card

Базовый класс для всех карточек товара.

Конструктор:
`constructor(container: HTMLElement, events: IEvents)` - принимает контейнер карточки и объект для обработки событий.

Поля класса:
`title: HTMLElement` - название товара.
`image: HTMLImageElement` - изображение товара.
`price: HTMLElement` - стоимость товара.
`category: HTMLElement` - категория товара.
`events: IEvents` - объект для обработки событий.

Методы класса:
`set title(value: string)` - отображает название товара.
`set image(value: string)` - отображает изображение товара.
`set price(value: number | null)` - отображает стоимость товара или значение "Бесценно", если цена отсутствует.
`set category(value: string)` - отображает категорию товара.

#### Класс CatalogCard

Наследуется от класса `Card`.
Отвечает за отображение карточки товара в каталоге, а также генерирует событие выбора товара при нажатии на карточку.

#### Класс PreviewCard

Наследуется от класса `Card`.
Отвечает за отображение карточки товара в модальном окне.

Дополнительные поля класса:
`description: HTMLElement` - описание товара.
`button: HTMLButtonElement` - кнопка покупки или удаления товара из корзины.

Методы класса:
`set description(value: string)` - отображает описание товара.
`set button(value: string)` - изменяет текст кнопки.
`set disabled(value: boolean)` - изменяет состояние кнопки.

#### Класс BasketCard

Наследуется от класса `Card`.
Отвечает за отображение товара в корзине.

Дополнительные поля класса:
`index: HTMLElement` - порядковый номер товара.
`button: HTMLButtonElement` - кнопка удаления товара.

Методы класса:
`set index(value: number)` - отображает порядковый номер товара.

#### Класс Basket

Отвечает за отображение содержимого корзины.

Конструктор:
`constructor(container: HTMLElement, events: IEvents)` - принимает контейнер корзины и объект для обработки событий.

Поля класса:
`list: HTMLElement` - список товаров.
`total: HTMLElement` - общая стоимость товаров.
`button: HTMLButtonElement` - кнопка оформления заказа.
`events: IEvents` - объект для обработки событий.

Методы класса:
`set items(items: HTMLElement[])` - отображает список товаров.
`set total(value: number)` - отображает общую сумму товаров в корзине.
`set disabled(value: boolean)` - изменяет состояние кнопки оформления заказа.

#### Класс Form<T>

Базовый класс для всех форм оформления заказа.
Является дженериком, принимает в параметре `T` тип данных формы.

Конструктор:
`constructor(container: HTMLFormElement, events: IEvents)` - принимает элемент формы и объект для обработки событий.

Поля класса:
`submit: HTMLButtonElement` - кнопка отправки формы.
`errors: HTMLElement` - элемент отображения ошибок.
`events: IEvents` - объект для обработки событий.

Методы класса:
`set valid(value: boolean)` - изменяет состояние доступности кнопки отправки формы.
`set errors(value: string)` - отображает сообщение об ошибке.
`render(state: Partial<T> & { valid: boolean; errors: string }): HTMLElement` - отображает состояние формы.
`reset(): void` - очищает значения полей формы.

#### Класс OrderForm

Наследуется от класса `Form`.
Отвечает за первый этап оформления заказа.
Также отображает кнопки выбора способа оплаты и поле ввода адреса доставки, управляет активным состоянием выбранного способа оплаты.

#### Класс ContactsForm

Наследуется от класса `Form`.
Отвечает за второй этап оформления заказа.
Также отображает поля ввода электронной почты и телефона.

#### Класс Success

Отвечает за отображение сообщения об успешном оформлении заказа.

Конструктор:
`constructor(container: HTMLElement, actions: { onClick: () => void })` - принимает контейнер компонента и обработчик закрытия окна.

Поля класса:
`description: HTMLElement` - элемент отображения итоговой стоимости заказа.
`button: HTMLButtonElement` - кнопка закрытия окна.

Методы класса:
`set total(value: number)` - отображает итоговую стоимость заказа.

`contacts:submit` - отправка формы контактов и оформление заказа.
`modal:open` - открытие модального окна.
`modal:close` - закрытие модального окна.

## Презентер

Роль презентера выполняет файл `main.ts`.

Основные задачи презентера:

- загрузка каталога товаров с сервера;
- создание и отображение карточек товаров;
- открытие и закрытие модальных окон;
- управление корзиной;
- обработка действий пользователя;
- передача данных между моделями и представлениями;
- валидация данных покупателя;
- отправка заказа на сервер;
- отображение сообщения об успешном оформлении заказа;
- сброс данных покупателя и форм оформления заказа.
